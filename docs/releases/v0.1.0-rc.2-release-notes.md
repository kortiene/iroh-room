## What Changed Since v0.1.0-rc.1

This candidate exists primarily to fix a cohort-blocking defect:

- **Join-after-conversation deadlock fixed (PR #111).** In rc.1, once any
  non-admin chat existed in a room, no new participant could complete
  `room join`: the invite cites the current DAG heads (chat events after a
  conversation), the membership fold requires every `prev_events` parent
  before classifying, `WantMembership` served only the bare authorization
  class, and the admin drops `WantEvents` backfill from provisional peers — a
  circular deadlock ending in a 10-second timeout blamed on an unreachable
  admin. `WantMembership` now serves the causal closure of the authorization
  class, and the requester claims every held event id, giving guaranteed
  multi-round bootstrap progress under the response cap. `room join` also
  distinguishes the new `membership_incomplete` error (admin responded, the
  membership history never completed this attempt) from `no_admin_reachable`.
- **Cross-room isolation hardened (PR #106).** Every event-id lookup in the
  sync engine is room-scoped; a row from another room in the shared store can
  no longer be served via `WantEvents`, satisfy a causal dependency, or clear
  the fail-closed admin-tip suspicion.
- **Relay-only test seam (PR #107).** Compile-time `relay-only-test` feature
  for Gate-A relay-throughput verification; off by default, no runtime switch.
- **Oversized-frame resilience (PR #111).** The peer writer drops a
  locally-queued frame that exceeds the 1 MiB frame cap with a warning instead
  of silently ending all sync to that peer.

See `crates/iroh-rooms/CHANGELOG.md` for the full entries.

## Upgrade Note — Mixed Versions Keep The Join Bug

An rc.1 **admin** still serves the bare authorization class, so a join whose
invite was minted after a conversation keeps failing even when the joiner runs
rc.2 (the joiner now at least reports `membership_incomplete` honestly).
**Every room member — especially the room admin — must upgrade to rc.2.** Do
not mix rc.1 and rc.2 in one room during the cohort.

## Supported Scope

This candidate supports:

- Identity creation.
- Private room creation.
- Key-bound room invites and joins — including joining a room after the
  conversation has started.
- Signed room messages.
- Verified file/artifact sharing.
- Authenticated loopback-only live TCP pipes with explicit per-member allow
  lists.
- Explicitly invited agent participants and signed `agent.status` updates.
- Best-effort local audit trail at `<IROH_ROOMS_HOME>/audit.ndjson`.

## Known Limitations

Read before installing or running this candidate:

- No central application server.
- No guaranteed offline message delivery.
- Local storage is plaintext.
- Invite tickets are password-grade capabilities.
- No native ticket-specific invite revocation; ADR-0002 accepts a bounded
  leaked-ticket model for beta scope only.
- While a join window (`room tail --accept-joins`) is open, a provisionally
  admitted dialer that has not proven invite possession can pull the
  membership bootstrap set, which may include chat that entered the membership
  ancestry (issue #112; pre-existing analogue: all admin-authored events).
- Rooms approaching ~30k events exceed the membership sync have-list frame
  budget; sync degrades with a logged warning rather than a dead link (issue
  #113). Far beyond cohort scale.
- A member device offline across a very long conversation (>64-message gap
  with no interleaved membership events) cannot catch up on chat (issue #114).
- `audit.ndjson` is local best-effort audit, not remote, centrally retained,
  tamper-evident, or compliance-grade audit.
- Binary artifacts are checksummed but not project-signed.
- SDK crate remains `publish = false`; SDK use is source/workspace-based for
  this candidate.
- Gate A evidence carries over from the 2026-07-07 refresh (no transport-layer
  behavior change in this candidate); the cellular relay-throughput caveat and
  missing home-NAT-to-CGNAT reverse-leg waivers remain in force.

## Security And Privacy Notes

- Storage posture: ADR-0001 scoped plaintext beta on trusted local machines.
- Invite posture: ADR-0002 bounded leaked-ticket model; no native invite
  revocation in this candidate.
- Audit posture: ADR-0003 local `audit.ndjson`; no central retention,
  tamper-evidence, or compliance claim.
- New in this candidate: the provisional join-bootstrap disclosure bound is
  documented in `specs/room-join-by-ticket.md` and tracked as issue #112;
  future hardening is capability-proof-gated bootstrap serving.
- Security reporting path: use the private maintainer channel for secrets,
  data loss, authorization bypass, or remote code execution concerns. Public
  bug reports must follow `.github/ISSUE_TEMPLATE/bug_report.md` and must not
  attach full tickets, `identity.secret`, `rooms.db`, blob contents,
  unredacted `audit.ndjson`, or full data-directory backups.
